# API Client — Instructions (openapi-fetch + openapi-typescript)

Skillexa Portal uses **openapi-fetch** (typed fetch client, 6 kB) backed by types generated with **openapi-typescript** CLI.

---

## Packages

| Package              | Type    | Purpose                                                 |
| -------------------- | ------- | ------------------------------------------------------- |
| `openapi-fetch`      | dep     | Lightweight typed fetch wrapper (runtime)               |
| `openapi-typescript` | devDep  | CLI that generates TypeScript types from OpenAPI schema |

---

## Generated Types Location

```
lib/
  api-client/
    schema.d.ts   ← generated by openapi-typescript CLI; commit this file; never edit manually
```

The `lib/api-client/` directory holds **only** `schema.d.ts`. There are no generated runtime files.

---

## Regenerating `schema.d.ts`

Run whenever the Core OpenAPI spec changes (Skillexa-Core must be running):

```bash
pnpm generate-api-client
```

This executes:

```bash
openapi-typescript http://localhost:8080/openapi/v1.json -o lib/api-client/schema.d.ts
```

If Core is unavailable, generate from a saved spec file:

```bash
openapi-typescript ./path/to/openapi.json -o lib/api-client/schema.d.ts
```

Commit `schema.d.ts` after regenerating.

## Client Factory (`lib/core-client.ts`)

```ts
import createClient, { type Middleware } from "openapi-fetch";
import type { paths } from "./api-client/schema";

export function createApiClient(accessToken: string) {
  const client = createClient<paths>({
    baseUrl: process.env.SKILLEXA_CORE_BASE_URL ?? "http://localhost:8080",
  });

  const authMiddleware: Middleware = {
    async onRequest({ request }) {
      request.headers.set("Authorization", `Bearer ${accessToken}`);
      return request;
    },
  };

  client.use(authMiddleware);
  return client;
}
```

- The public signature `createApiClient(accessToken: string)` is fixed — do not change it.
- `accessToken` is the server-minted Portal JWT for Core, not a Microsoft or Google provider token.
- `baseUrl` is read from `process.env.SKILLEXA_CORE_BASE_URL`; falls back to `http://localhost:8080` for local dev.

## Request Patterns

### GET with path and query params

```ts
const { data, error } = await client.GET("/documents/{id}", {
  params: {
    path: { id: documentId },
    query: { page: 1 },
  },
});
```

### POST with a request body

```ts
const { data, error } = await client.POST("/documents", {
  body: { jobTitle: "Software Engineer", templateKey: "default" },
});
```

### Other HTTP methods

```ts
await client.PUT("/admin/templates/{id}", { params: { path: { id } }, body: { ... } });
await client.DELETE("/admin/templates/{id}", { params: { path: { id } } });
```

### Response shape

Every call returns `{ data, error, response }`:

| Field      | Present when        | Content                                       |
| ---------- | ------------------- | --------------------------------------------- |
| `data`     | 2xx response        | Typed response body                           |
| `error`    | 4xx / 5xx / default | Typed error body                              |
| `response` | always              | Raw `Response` (status, headers, etc.)        |

**Always check `error` before using `data`:**

```ts
const { data, error } = await client.GET("/app/usage");
if (error) return new NextResponse("Service unavailable", { status: 502 });
return NextResponse.json(data ?? {});
```

## Endpoint Reference

| Method | Path                           | Call pattern                                                                 |
| ------ | ------------------------------ | ---------------------------------------------------------------------------- |
| POST   | `/provision`                   | Used by sign-in provisioning with a bootstrap Core JWT                        |
| POST   | `/job-listings/search`         | `client.POST("/job-listings/search", { body })`                              |
| POST   | `/documents`                   | `client.POST("/documents", { body })`                                        |
| GET    | `/documents`                   | `client.GET("/documents")`                                                   |
| GET    | `/documents/{id}`              | `client.GET("/documents/{id}", { params: { path: { id } } })`                |
| POST   | `/documents/{id}/download-url` | `client.POST("/documents/{id}/download-url", { params: { path: { id } } })` |
| GET    | `/app/usage`                   | `client.GET("/app/usage")`                                                   |

## Server-Side Only Rule

The client is used **exclusively on the server side**:

- ✅ Route Handlers (`app/api/*/route.ts`)
- ✅ Server Components (`page.tsx`, `layout.tsx`)
- ✅ Server Actions

**Never import the client in client components.** Client components call BFF route handlers (`/api/*`) which forward to Core.

---

## Authentication Pattern (Route Handlers)

```ts
import { createApiClient } from "@/lib/core-client";
import { createCoreAccessToken } from "@/lib/auth/core-token";

const accessToken = await createCoreAccessToken(request);
if (!accessToken) return new NextResponse("Unauthorized", { status: 401 });
const client = createApiClient(accessToken);

const { data, error } = await client.GET("/documents");
```

---

## Best Practices

- Always use literal URL strings — never build paths dynamically at runtime (e.g., `"/documents/{id}"` not `"/documents/" + id`).
- Always destructure `{ data, error }` and check `error` before using `data`.
- Path parameters go in `params.path`, query parameters in `params.query`, request bodies in `body`.
- Do **not** manually edit `lib/api-client/schema.d.ts` — it is generated output.
