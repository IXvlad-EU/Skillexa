# When to Use

- Add UI text to a new or existing component, page, or feature.
- Create or update message files (`en.json`, `ru.json`, `de.json`).
- Set up `next-intl` for the first time or fix misconfigured locale routing.
- Use `useTranslations` / `getTranslations` in a Server or Client Component.
- Generate locale-aware `<Link>` / `redirect` / `useRouter` via `i18n/navigation.ts`.
- Add `generateMetadata` with translated page titles.
- Pass translated strings to Mantine components.

---

## Golden Rule

**No hardcoded user-facing text anywhere in the codebase.** Every string rendered to the user — labels, buttons, headings, placeholders, error messages, tooltips, ARIA attributes, notifications, confirmation dialogs — must come from a translation message file.

## File Structure

```
skillexa-portal/
  i18n/
    navigation.ts     # locale-aware Link, redirect, useRouter, usePathname
    request.ts        # getRequestConfig — loads locale + messages on every request
    routing.ts        # defineRouting — locales, defaultLocale
  messages/
    en.json           # source of truth — add keys here first
    ru.json           # must mirror en.json keys exactly
    de.json           # must mirror en.json keys exactly
  proxy.ts            # next-intl middleware (Next.js 16 uses proxy.ts, not middleware.ts)
  next.config.ts      # withNextIntl plugin
  app/
    [locale]/
      layout.tsx      # NextIntlClientProvider, html lang={locale}, locale validation
```

## Configuration

### `i18n/routing.ts`

```ts
import { defineRouting } from 'next-intl/routing';

export const routing = defineRouting({
  locales: ['en', 'ru', 'de'],
  defaultLocale: 'en',
});
```

### `i18n/request.ts`

```ts
import { getRequestConfig } from 'next-intl/server';
import { hasLocale } from 'next-intl';
import { routing } from './routing';

export default getRequestConfig(async ({ requestLocale }) => {
  const requested = await requestLocale;
  const locale = hasLocale(routing.locales, requested)
    ? requested
    : routing.defaultLocale;

  return {
    locale,
    messages: (await import(`../messages/${locale}.json`)).default,
  };
});
```

### `i18n/navigation.ts`

```ts
import { createNavigation } from 'next-intl/navigation';
import { routing } from './routing';

export const { Link, redirect, usePathname, useRouter, getPathname } =
  createNavigation(routing);
```

### `proxy.ts` (root of `skillexa-portal/`)

```ts
import createMiddleware from 'next-intl/middleware';
import { routing } from './i18n/routing';

export default createMiddleware(routing);

export const config = {
  matcher: '/((?!api|trpc|_next|_vercel|.*\\..*).*)',
};
```

### `next.config.ts`

```ts
import createNextIntlPlugin from 'next-intl/plugin';
import type { NextConfig } from 'next';

const withNextIntl = createNextIntlPlugin('./i18n/request.ts');

const nextConfig: NextConfig = { output: 'standalone' };

export default withNextIntl(nextConfig);
```

## Message File Format

Organize messages into **namespaces** matching feature areas:

```jsonc
// messages/en.json
{
  "common": {
    "appName": "Skillexa",
    "loading": "Loading…",
    "save": "Save",
    "cancel": "Cancel",
    "error": "Something went wrong",
    "retry": "Retry"
  },
  "auth": {
    "login": "Log in",
    "logout": "Log out",
    "loginButton": "Sign in"
  },
  "jobs": {
    "title": "My Jobs",
    "status": {
      "pending": "Pending",
      "processing": "Processing",
      "succeeded": "Succeeded",
      "failed": "Failed"
    },
    "download": "Download PDF",
    "noJobs": "You have no jobs yet"
  },
  "documents": {
    "title": "Create Document",
    "submit": "Generate"
  },
  "validation": {
    "required": "{field} is required",
    "minLength": "{field} must be at least {min} characters",
    "email": "Enter a valid email address"
  }
}
```

### Message Rules

- `en.json` is the **source of truth** — add keys here first, then mirror to `ru.json` and `de.json`.
- All three files must have **identical key structures** at all times.
- Use **ICU interpolation** for dynamic values: `"greeting": "Hello, {name}!"`.
- Use **ICU plural rules**: `"itemCount": "You have {count, plural, =0 {no items} one {# item} other {# items}}"`.
- Use **rich text** when needed: `"terms": "By signing up you agree to our <link>Terms</link>"`.
- Namespace names match feature folders: `auth`, `jobs`, `documents`, `common`, `validation`.

## Usage Patterns

### Server Component (default)

```tsx
import { useTranslations } from 'next-intl';

export default function JobsPage() {
  const t = useTranslations('jobs');
  return <h1>{t('title')}</h1>;
}
```

### Client Component

```tsx
'use client';
import { useTranslations } from 'next-intl';

export function SubmitButton() {
  const t = useTranslations('documents');
  return <Button type="submit">{t('submit')}</Button>;
}
```

### `generateMetadata`

```tsx
import { getTranslations } from 'next-intl/server';

export async function generateMetadata({ params }: { params: Promise<{ locale: string }> }) {
  const { locale } = await params;
  const t = await getTranslations({ locale, namespace: 'jobs' });
  return { title: t('title') };
}
```

### Locale-Aware Navigation

```tsx
import { Link } from '@/i18n/navigation';
<Link href="/">Skillexa</Link>
```

### Mantine Components

Always pass translated strings — never raw text:

```tsx
const t = useTranslations('auth');
<TextInput label={t('email')} placeholder={t('emailPlaceholder')} />
<Button>{t('loginButton')}</Button>
```

## Root Locale Layout

`app/[locale]/layout.tsx` must:
1. Receive `locale` from `params`.
2. Validate with `hasLocale` — call `notFound()` if invalid.
3. Set `<html lang={locale}>`.
4. Wrap children with `<NextIntlClientProvider>`.

```tsx
import { NextIntlClientProvider, hasLocale } from 'next-intl';
import { notFound } from 'next/navigation';
import { routing } from '@/i18n/routing';

export default async function LocaleLayout({
  children,
  params,
}: {
  children: React.ReactNode;
  params: Promise<{ locale: string }>;
}) {
  const { locale } = await params;
  if (!hasLocale(routing.locales, locale)) notFound();

  return (
    <html lang={locale}>
      <body>
        <NextIntlClientProvider>{children}</NextIntlClientProvider>
      </body>
    </html>
  );
}
```
